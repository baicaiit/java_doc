设计模式是前人为了解决特定问题实践得到的解决方案

# 软件设计原则

  • 开闭原则（Open Close Principle）

​ 就是说对扩展开放，对修改关闭

  • 里氏替换原则（Liskov Substitution Principle）

​ 子类继承父类时，除添加新的方法完成新增功能外，尽量不要重写父类的方法

  • 依赖倒转原则（Dependence Inversion Principle）

    要面向接口编程，不要面向实现编程

  • 接口隔离原则（Interface Segregation Principle）

    降低依赖，降低耦合

  • 迪米特法则（最少知道原则）（Demeter Principle）

    降低类之间的耦合度，提高模块的相对独立性

  • 合成复用原则（Composite Reuse Principle）

    尽量使用合成/聚合的方式，而不是使用继承, 如果要使用继承关系，则必须严格遵循里氏替换原则

  • 单一原则

    一个类只做一件事情

# 模板方法模式

定义了一个操作中的算法骨架，而将算法的一些步骤延迟到子类中执行 -> (类似插槽)

写一个例子:

一个人的一天，起床 吃饭 劳动 吃饭 劳动 看新闻 吃饭 睡觉

  • 定义模版
abstract class Human {

  protected String name;

  /**
     * 劳动的方法
     */
  abstract void work();

  /**
     * 吃的方法
     */
  abstract void eat();

  /**
     * 看新闻
     */
  abstract void readNews();

  /**
     * 一天的生活
     */
  public void life(){
    eat();
    work();
    eat();
    work();
    eat();
    readNews();
  }
}

public class Star extends Human{

  public Star(String name) {
    this.name = name;
  }

  @Override
  void work() {
    System.out.println(this.name + "一直很烦恼的干活！");
  }

  @Override
  void eat() {
    System.out.println(this.name + "吃不下饭");
  }

  @Override
  void readNews() {
    System.out.println(this.name + "一边看新闻，一边想明天的饭菜");
  }

}

# 策略模式

对象数组排序怎么排序? 要排序就要有两两比较的过程

class User {
    public String username;
    public Integer age;
    public Double height;

    public User(String username, Integer age, Double height) {
        this.username = username;
        this.age = age;
        this.height = height;
    }

    @Override
    public String toString() {
        return "User{" +
                "username='" + username + '\'' +
                ", age=" + age +
                ", height=" + height +
                '}';
    }
}

public class Client {
    public static void main(String[] args) {
        User user1 = new User("小王", 12, 153.4);
        User user2 = new User("小李", 14, 163.4);
        User user3 = new User("小张", 13, 123.4);
        User user4 = new User("小杨", 4, 6.4);

        User[] users = { user1, user2, user3, user4 };
        // 冒泡排序 根据age来排序
        for (int i = 0; i < users.length - 1; i++) {
            for (int j = 0; j < users.length - i - 1; j++) {
                if (users[j].age > users[j + 1].age) {
                    User temp = users[j];
                    users[j] = users[j + 1];
                    users[j + 1] = temp;
                }
            }
        }

        // 输出排序后结果
        for (int i = 0; i < users.length; i++) {
            System.out.println(users[i]);
        }
    }
}

interface Comparable {
    // Integer compare(Object obj);
    Integer compare(Comparable comparable); // 让其有比较能力
}

class User implements Comparable {
    public String username;
    public Integer age;
    public Double height;

    public User(String username, Integer age, Double height) {
        this.username = username;
        this.age = age;
        this.height = height;
    }

    @Override
    public String toString() {
        return "User{" +
                "username='" + username + '\'' +
                ", age=" + age +
                ", height=" + height +
                '}';
    }

    @Override
    public Integer compare(Comparable obj) {
        if (obj instanceof User) {
            User user = (User) obj;
            return this.age - user.age;  // 如果不想比较年龄或修改排序, 在这里改就行 // 要怎么比, 对象来决定, 对外屏蔽掉
        }
        return null;
    }
}

想要不同的比较方式, 对象来决定, 细节对外屏蔽掉

但上面的写法还是不友好的, 因为想修改比较的内容, 就必须修改User类，这很明显违反了开闭原则

interface Comparable {
    // Integer compare(Object obj);
    // Integer compare(Comparable comparable); // 让其有比较能力
    int compare(User user1, User user2);
}

// 根据年龄的比较器 
class UserAgeComparator implements Comparable {
    @Override
    public int compare(User user1, User user2) {
        return user1.age - user2.age;
    }
}

class SortUtil {
    public static void sort(User[] users, Comparable obj) {
        // 冒泡排序
        for (int i = 0; i < users.length - 1; i++) {
            for (int j = 0; j < users.length - i - 1; j++) {
                if (obj.compare(users[j], users[j + 1]) > 0) {
                    User temp = users[j];
                    users[j] = users[j + 1];
                    users[j + 1] = temp;
                }
            }
        }
    }
}

class User {
    public String username;
    public Integer age;
    public Double height;

    public User(String username, Integer age, Double height) {
        this.username = username;
        this.age = age;
        this.height = height;
    }

    @Override
    public String toString() {
        return "User{" +
                "username='" + username + '\'' +
                ", age=" + age +
                ", height=" + height +
                '}';
    }
}

public class Client {
    public static void main(String[] args) {
        User user1 = new User("小王", 12, 153.4);
        User user2 = new User("小李", 14, 163.4);
        User user3 = new User("小张", 13, 123.4);
        User user4 = new User("小杨", 4, 6.4);

        User[] users = { user1, user2, user3, user4 };

        // 比较代码不暴露在使用者处了, 转移到了工具类中
        // user类也不用改了, 想要别的比较方式, 换其他比较器就行
        SortUtil.sort(users, new UserAgeComparator()); 

        // 输出排序后结果
        for (int i = 0; i < users.length; i++) {
            System.out.println(users[i]);
        }
    }
}
